# 더나은세무회계 — 종합소득세 신고대리 랜딩 & 접수 폼 (샘플)

## 파일 구성
- `index.html` — 랜딩 페이지 + 1분 진단 + 스텝형 접수 폼 (단일 HTML)
- `Code.gs`    — 구글 스프레드시트에 붙일 Apps Script 코드
- `README.md`  — 이 문서

## 현재 상태
샘플은 **데모 모드**로 실행됩니다 (`CONFIG.DEMO_MODE = true`).
- 폼 제출 시 서버로 전송되지 않고 완료 모달만 뜸
- 브라우저 콘솔(F12)에서 실제 전송될 payload 확인 가능
- 카카오톡 버튼은 placeholder URL로 이동

## 구글 스프레드시트 연동 설정 (5분)

### 1) 스프레드시트 생성
Google Drive에서 새 스프레드시트 생성 (예: `더나은_2026종소세_접수`).

### 2) Apps Script 등록
① 시트에서 메뉴 **확장 프로그램 → Apps Script** 클릭
② 기본 코드 삭제하고 `Code.gs` 전체 내용 복사·붙여넣기
③ 저장 (Ctrl+S), 프로젝트 이름 자유롭게 지정

### 3) 웹 앱으로 배포
① Apps Script 우측 상단 **배포 → 새 배포** 클릭
② 유형 선택 ⚙ → **웹 앱** 선택
③ 설정:
   - **설명**: 접수폼 수신기 v1
   - **다음 사용자 인증으로 실행**: 나 (본인 구글 계정)
   - **액세스 권한**: **모든 사용자** (익명 접근 허용)
④ **배포** 클릭 → 권한 허용 (Drive, Sheets, Mail 권한 요청됨)
⑤ 발급된 **웹 앱 URL** 복사 (`https://script.google.com/macros/s/.../exec`)

### 4) index.html 설정
`index.html` 상단 스크립트에서 아래 값 수정:

```js
const CONFIG = {
  GAS_ENDPOINT: "https://script.google.com/macros/s/..../exec", // ← 3번에서 복사한 URL
  KAKAO_URL:    "https://pf.kakao.com/_YOURID/chat",             // ← 카카오 채널 URL
  DEMO_MODE:    false                                            // ← false로 변경
};
```

### 5) 테스트
- index.html을 로컬 또는 웹서버에서 열어 폼 제출
- 스프레드시트에 행이 추가되고, Drive `더나은_접수파일` 폴더에 업로드 파일이 저장되는지 확인

## ⚠️ 보안 설계 요약

### 시트에 저장되지 않는 민감정보
- **주민등록번호 뒷 6자리** → `******`로 마스킹 후 저장. 완전한 번호는 세무사가 별도 채널(전화/카톡)로 확인
- **홈택스 비밀번호** → 시트에 저장 안 됨. 제출 여부(Y/N)만 기록. 실제 비밀번호는 세무사에게 별도 전달
- **부양가족 주민번호 뒷자리도 동일 마스킹**

### 추가 권장 보안 조치 (운영 시)
1. 스프레드시트 **공유 범위를 담당자 이메일로 최소화**
2. Drive 폴더 링크 공유 설정을 "링크 있는 모두" → "특정 사용자"로 변경
3. Apps Script 액세스 권한을 "모든 사용자" → "도메인 내 모든 사용자"로 좁히기 (Google Workspace 사용 시)
4. HTTPS 도메인에 호스팅 (예: Cloudflare Pages, Netlify, Vercel)
5. 운영 단계에서는 자체 서버 + DB 암호화(AES-256) 권장

### 세무사 이메일 알림 (선택)
`Code.gs`의 `_handleSubmission` 함수 마지막 부분의 주석을 해제하고 이메일을 입력하면 접수 즉시 이메일로 알림이 옵니다:
```js
_notifyEmail('your@email.com', d);
```

## 커스터마이징 포인트

- **대표 프로필 사진**: `.profile-photo` div를 `<img src="...">`로 교체
- **로고**: `.brand .mark` 구조 부분을 `<img src="logo.png">`로 교체
- **전화번호·주소·이메일**: 푸터(`<footer>`) 내용 수정
- **수수료/할인 정보**: FAQ 답변 텍스트 수정
- **카카오 채널 URL**: `CONFIG.KAKAO_URL` 수정
- **도메인**: `<link rel="canonical">` 와 JSON-LD `"url"` 필드에 실제 도메인 기입 (SEO용)

## SEO 최적화 반영 항목

- 메타 태그 (title, description, keywords, OG)
- FAQ 구조화 데이터 (Schema.org JSON-LD) — 구글 검색 FAQ 리치결과 노출
- LocalBusiness 구조화 데이터 — 지역 검색 최적화 (부산 금정구 장전동)
- 케이스별 롱테일 키워드 질문문 (자영업/프리랜서/임대) — 실제 검색어와 매칭
- 랜딩 페이지 형태 단일 HTML (크롤링 친화적)
